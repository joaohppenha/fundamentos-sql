# <span style="font-size:20pt"><b>Fundamentos de SQL</b></span>

## Sobre o Projeto

Este projeto foi desenvolvido para demonstrar conhecimentos fundamentais de **SQL** aplicados em um cenário real de **gestão escolar**. O objetivo foi criar um banco de dados completo com tabelas de alunos, professores, disciplinas, turmas, matrículas e avaliações, utilizando **BigQuery** como ambiente de execução.  

O projeto foi dividido em cinco partes principais, cada uma representando uma categoria de comandos SQL:  

---

## **1. DDL (Data Definition Language)**

Nesta etapa, foram aplicados conceitos de **criação e alteração de estruturas de banco de dados**, incluindo:  

- Criação de **dataset** e **tabelas**.  
- Alterações em colunas e adição de descrições.  
- Demonstração de comandos como `CREATE TABLE`, `ALTER TABLE` e `DROP TABLE`.  

Esse módulo demonstra o conhecimento em **modelagem de dados** e preparação de um banco para uso real.  

---

## **2. DML (Data Manipulation Language)**

Aqui o foco foi na **manipulação de dados**, aplicando comandos como:  

- `INSERT` para adicionar registros de alunos, professores, disciplinas, turmas, matrículas e avaliações.  
- `UPDATE` para modificar informações existentes.  
- `DELETE` para remoção de registros específicos.  
- `TRUNCATE` para limpeza completa de tabelas mantendo a estrutura.  

Este módulo evidencia habilidades em **gestão de dados e consistência** dentro do banco.  

---

## **3. DQL (Data Query Language)**

O objetivo do DQL é realizar **consultas e análises de dados**. Foram aplicados:  

- `SELECT` simples e com filtros (`WHERE`).  
- Agregações com `GROUP BY` e `HAVING`.  
- Ordenações com `ORDER BY`.  
- `JOINs` para relacionar múltiplas tabelas e extrair informações integradas.  

Esse módulo demonstra **capacidade de análise e extração de insights** dos dados.  

---

## **4. DCL (Data Control Language)**

O DCL foca em **controle de permissões e segurança de dados**, utilizando:  

- `GRANT` para conceder permissões de leitura, escrita ou administração.  
- `REVOKE` para remover permissões.  
- Controle em nível de **dataset** e **tabela**.  

Este módulo demonstra **entendimento de segurança e governança de dados**.  

---

## **5. DTL (Data Transaction Language)**

O DTL permite **controle de transações**, garantindo integridade e consistência:  

- `BEGIN TRANSACTION` para iniciar uma transação.  
- `COMMIT` para confirmar alterações.  
- `ROLLBACK` para reverter alterações em caso de erros.  

Este módulo evidencia **controle de atomicidade e confiabilidade** em operações críticas.  

---

## **Tecnologias Utilizadas**

- **SQL**: Linguagem padrão para manipulação, definição, consulta e controle de dados.  
- **BigQuery**: Ambiente de execução em nuvem para armazenamento e análise de grandes volumes de dados.  
- **Google Cloud Platform (GCP)**: Para gerenciamento de datasets e exportação de resultados.
